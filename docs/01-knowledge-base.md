## 0. 中核思想（すべての前提）

> **「金融機械学習（Financial ML）」は「金融へのMLの適用（ML applied to Finance）」ではない。**

- 汎用ML（顔認識・LLM等）は **IID・高S/N比** を前提とする。金融データは **非定常・強シリアル相関・低S/N比・非IID** であり、その前提が根本から崩れる。
- ゆえに金融専用のツールキットと研究プロセスが必要。MLは **ブラックボックスではなくホワイトボックス（研究ツール）** として使う。
- 成功の鍵は新しいアルゴリズムではなく、**科学的厳密性**（データ構造化・ラベリング・検証・多重検定制御）。これらは**モデル非依存**（Transformerでも変わらない）。
- アルファ抽出は属人芸ではなく**工業化された組立ライン**で行う（マンハッタン計画の比喩）。

## 1. 問題認識：なぜ大半のMLファンドが失敗するか

| # | 問題 | 本質 |
|---|------|------|
| P1 | **多重検定／選択バイアス (SBuMT)** | 良い結果が出るまで試行を繰り返し、勝者だけを提示。試行数を隠すと偽陽性確率を評価不能。 |
| P2 | **バックテスト過学習** | 低S/N比ゆえ「適合」より「過剰適合」の確率が圧倒的に高い。ノイズから利益を得ているのに信号に見える。 |
| P3 | **金融は適応系** | 発見した効果は裁定で消える。「Newtonの『プリンキピア』出版が重力を打ち消す」世界。観測が対象を最も都合悪く変質させる。 |
| P4 | **実験室の不在** | フラッシュクラッシュを統制条件下で再現できない。検証は過学習に使ったのと同じ時系列のみ。真のアウトオブサンプルは数十年かかる。 |
| P5 | **低S/N比** | 信号がノイズとほぼ識別不能。偽発見を避けるのも真発見を捨てないのも極めて困難。 |

**帰結**：「金融経済学の主張の多くは偽である可能性が高い」（Harvey, AFA会長講演）。過去70年の実証研究の相当部分が無効化されうる。

## 2. 組織・プロセス設計（柱B-org）

### サイロ型 → 研究所/組立ライン型
- ❌ **シロ型**：1人のクオンツがデータ清掃〜バックテストまで全工程を孤立して担当 →「シシュポス的クオンツ」（再現性のない岩を押し上げ続ける徒労）。過学習戦略や既知の低収益戦略を再生産。
- ✅ **組立ライン型**：工程を専門分化し、科学的対話で統合。

| 工程 | 担当領域 |
|------|----------|
| データ・キュレーション | 情報バー構築、分数階差分、クレンジング |
| 特徴量エンジニアリング | 理論ベースの特徴量、メタラベル設計 |
| 戦略開発 | アルゴリズム選択、ラベリング |
| バックテスト検証（監査） | パージング/エンバーゴ、CPCV、DSR |
| ポートフォリオ構築 | 配分、執行コスト最適化 |
| 執行 | マイクロ構造、注文執行 |

### ガバナンス
- **研究監視 (Research Surveillance)**：全試行を自動記録。研究者は試行を削除・選別できない。データアクセス回数に応じて期待値をペナルティ化（DSRに反映）。
- **査読的検証役**：研究者とは別に、結果に疑義を呈し検証する役割を組織内に常設。
- **試行の「完全・強制・無汚染」(Complete / Coerced / Untainted)**：全試行を記録、研究者は何を残すか選べない、外部試行（事前選択済みでバイアスあり）を混入させない。

### 規模の住み分け（ライオンとトカゲ）
- 大規模ラボ（〜100億ドル）は工業化が必須だが、小アルファ（例：1日1万ドル）は狙わない。
- 小規模・専門特化プレイヤーには十分な機会が残る。大口執行が生む一時的非効率が小口の裁定機会になる**共生関係**。

## 3. データ前処理

### 3.1 分数階差分（Fractional Differentiation）— メモリ vs 定常性
- 過去80年の金融論文はほぼ全て **d=1の整数差分（リターン化）** で定常化 → **予測の源泉である「メモリ」を完全抹消**。
- **0か1かのジレンマ**：d=0（原系列）はメモリ最大だが非定常／d=1（リターン）は定常だがメモリ消滅（ランダムウォーク化）。
- 解：**0と1の中間のd**で、定常性を確保しつつメモリを最大保持。

| d | ADF統計量 | 原系列との相関 | 判定 |
|---|-----------|----------------|------|
| 0.0 | ≈0（非定常） | 1.000 | メモリ最大・学習不可 |
| **0.35** | **−2.86（定常）** | **≈0.995** | **最適** |
| 1.0 | ≈−45（過剰定常） | ≈0.05 | メモリ抹消・失格 |

- 主要先物の多くは **d<0.5** で定常化。S&P500等は **d≈0.4** で定常かつ相関>0.99。
- **EMHの再解釈**：市場が予測不能に見えるのは、**過剰差分が生んだ数学的アーティファクト**であって市場の本質ではない可能性。
- 実装要件：ADF検定を通す最小のdを動的に算出（相関最大化）。

### 3.2 情報主導型サンプリング — 時間軸からの脱却
- ❌ **時間バー**：情報到着率は一定でないのに均等にサンプリング → 不等分散・系列相関が悪化、夜間の冗長点と高密度変化の見逃し。
- 代替（情報密度に同期）：

| バー種別 | 抽出基準 |
|----------|----------|
| ティックバー | 一定の取引回数ごと |
| ボリュームバー | 一定の取引数量ごと |
| **ドルバー** | 一定の取引代金ごと（**標準推奨**） |
| ドル・インバランスバー | オーダーフロー不均衡を監視 |

- **ドルバーの優位**：資産価格変動・FIXプロトコル変更・ロットサイズ細分化に頑健、最も安定した統計特性。情報流入に応じ頻度が自動調整。
- **インフォームド・トレーダー検知**：情報優位が即時執行コストを上回ると判断した者が「スプレッドを跨いで」成行執行する非対称行動こそ捕捉すべき信号。**VPIN**（Volume-synchronized Probability of Informed Trading）で流動性ショックの予兆を捉える。

## 4. ラベリングとサンプル設計

### 4.1 トリプルバリア法
固定ホライズン（「n日後のリターン」）はパス依存性を無視した空論。3つの境界の**最初に触れたもの**をラベルとする：
1. **上方バリア**（利確）
2. **下方バリア**（損切り）
3. **垂直バリア**（時間切れ）

→ 「10日後は上昇でも途中でストップアウト」のような現実を反映。

### 4.2 メタラベリング（補正的AI / Corrective AI）
- **二段構造**：①売買の「方向」を決める一次モデル（人間/ファンダ/既存ロジックでも可）→ ②「その賭けに乗るか・サイズ」を決める二次MLモデル。
- 二次モデルの入力に**一次予測の確率分布・不確実性**を含める。
- 効果：**適合率（Precision）を劇的に改善**し偽陽性（空振り）を排除。「高Recall・低Precision」なモデル（不況を実際の倍予測する経済学者）を救済。Recall低下を容認し、確度の高い機会に資本集中。

### 4.3 サンプル独自性（非IID対処）
- 隣接観測はラベル評価期間が重なり、ほぼ同一情報を共有。
- 各サンプルの**独自性（Uniqueness）**を測定し、重複の多いサンプルの重みを下げる → アウトオブサンプル精度が劇的改善。

## 5. 検証 — バックテストの科学化

### 5.1 リーク防止：パージング & エンバーゴ
非IIDゆえ訓練/テスト境界で情報が漏洩する。標準k-fold CVは無価値化する。
- **パージング (Purging)**：テストセットとラベル期間が時間的に重なる訓練観測を削除。
- **エンバーゴ (Embargo)**：テスト直後の訓練観測を一定期間削除（残存シリアル相関を遮断）。
- → バックテストSharpeは下がるが、それが「現実的なアルファ」への接近。

### 5.2 CPCV（組合せパージ済交差検証）
単一パス検証を排し、複数シミュレーションパスを生成 → Sharpeを**点推定でなく分布**として評価し過学習リスクを統計的に制御。

### 5.3 デフレ・シャープレシオ（DSR）
「データを十分に拷問すれば何でも自白する」。試行数の報告なきSharpeは無価値。

ランダムウォークでも試行を繰り返せば高Sharpeが偶然出現：

| 試行回数 | 期待される最大Sharpe |
|----------|----------------------|
| 1,000 | ≈3.0 |
| 1,000,000 | ≈5.0 |

**DSRが補正する4要素**：①多重検定/選択バイアス（試行数K）②非正規性（歪度・尖度）③系列相関（メモリ効果）④試行間分散（**最重要かつ論文で決して報告されない変数**）。

- **False Strategy 定理**：試行数と試行間Sharpe分散から **E[max SR]**（偶然で到達する最大Sharpeの期待値）を導出。観測Sharpeがこれを有意に超えるかで真偽を判定。
- DSR = 多重検定等を補正した後に「真のSharpe>0」である確率。

### 5.4 試行のクラスタリング（実質独立試行数）
- 相関の高い試行を1系列に束ね（最小分散配分で外れ値支配を防止）、**実質的に独立なクラスタ数**を求める → これが真の「試行数」。
- 理論的制約の強い探索は少数クラスタに収まる（恣意的探索は数百クラスタに膨張）。

## 6. ポートフォリオ構築

### 6.1 マークウィッツ批判：「誤差最大化ツール」
共分散行列の逆行列×期待リターンが、2つの不安定性で破綻：
- **ノイズ誘導不安定性**：観測数<変数数だと共分散にノイズ過多。RMTによれば**固有値の約99%はノイズ**。
- **信号誘導不安定性**：高相関（多重共線性）で行列式が極小化→逆行列係数が肥大→入力の微小変化で最適ウェイトが激変。
- 「将来の共分散と期待リターンを完璧に知る」という不可能な前提に立つGIGO。

### 6.2 解決手法

| 手法 | 概要と効果 |
|------|-----------|
| **ノイズ除去 (Denoising)** | RMT（Marčenko-Pastur分布）でノイズ固有値を平均で置換。平均分散PFのMSEを**60%削減**、最大Sharpe PFで**94%削減**。 |
| **HRP（階層的リスクパリティ）** | 資産関係を樹状構造で捉え逆行列不要。必要データを5年分→2ヶ月分に短縮、Sharpeを約1/3改善。 |
| **NCO（ネスト型クラスター最適化）** | 高相関ブロックに分割して最適化。信号誘導不安定性をブロック内に封じ込め全体波及を防止。 |
| **アンサンブル** | バギング/ブースティング/スタッキング。**MLにおける数少ない「フリーランチ」**。分散を抑え過学習を回避、単一故障点を排除。 |

## 7. 因果推論レイヤー（ファクター投資の堅牢化・柱C）

### 7.1 予測 vs 帰属（Association vs Attribution）
| | 予測 (Forecasting) | 因果帰属 (Factor Investing) |
|---|---|---|
| 目的 | 関連性の収益化（ペアトレード等） | リスク源泉へのプレミアム帰属 |
| 因果の特定 | 二次的 | **必須** |
| 手法流用の危険 | — | 予測ツールを帰属に流用＝幻覚に基づくリスク管理 |

### 7.2 交絡因子 vs 合流点（コライダー）
| 変数タイプ | 構造 | 処理 |
|-----------|------|------|
| **交絡因子 (Confounder)** | X ← Z → Y（共通の原因） | **必ずモデルに含める**（除外すると偽相関） |
| **合流点 (Collider)** | X → Z ← Y（共通の結果） | **絶対に含めない**（含めると偽相関を創出） |

### 7.3 コライダーバイアスと符号逆転
- 真モデル `y = xβ + ε`、合流点 `z = yγ + xδ`。R²目当てにzを回帰に入れると：

  **E[β̂] = β − δγ / (1 + γ²)**

- バイアスが十分大きいと**正のプレミアムが負の推定値に「符号逆転」**＝買いシグナルが売りシグナルとして提示される。
- **非収益化**：ファクターモデルは同期間（t=t）観測の帰属モデル。合流点はリターンの「結果」なので、観測時点でリターンは既に実現済み＝投資収益として享受不能。

### 7.4 「General-to-Simple」/「キッチンシンク」回帰の罠
- 調整済みR²最大化・p値最小化のため全変数を投入する慣行が、コライダー混入リスクを極大化。統計指標の改善がモデルの正しさを保証するという盲信が損失を隠蔽。

### 7.5 因果探索とDAG
- **DAG（有向非巡回グラフ）を必須提示**。ドメイン知識で方向性を先行定義 ＋ データ駆動で検証：
  - **PCアルゴリズム**：条件付き独立性テストでグラフ構造推定
  - **GES**：適合度を最適化するエッジを段階探索
  - **LiNGAM**：非ガウス性を利用し因果方向を一意特定
- 注：金融のノイズ下ではDeep系因果探索は機能しにくく、古典手法を優先。
- **上流＝潜在的交絡因子（考慮）／下流＝潜在的合流点（排除）**。
- ⚠️ Granger因果（時間的先行性）は不十分：潜在変数とプロキシのタイムラグ、集計データの前後関係曖昧化で方向を誤示唆しうる。

### 7.6 モデル・リネージと開示
- DAG・各変数の採用/除外論理の明示を**義務化**。**DAGなき回帰は本番デプロイ資格なし**。
- レビュー観点を「P値・R²の確認」から「この変数は実は合流点では？／重要な交絡因子の欠落（Omitted variable bias）は？」へシフト。

## 8. ML応用10分野（資産運用全体像）

1. ポートフォリオ構築（HRP/NCO）
2. ベットサイジング（Precision/Recallベース）
3. 構造的変化の特定（相関構造の境界検知）
4. 堅牢な推定（RANSAC等で外れ値耐性）
5. ヘッジ（ディープ・ヘッジ）
6. 非構造化データ活用（センチメント）
7. 特徴量重要度（可読性・脱ブラックボックス）
8. 虚偽戦略の検知（DSR）
9. 注文執行（マイクロ構造）
10. 格付け自動化（フォーリング・エンジェル前のポジション構築）

> **価格予測はMLの最も信頼性が低く興味の薄い用途。** 真価はポートフォリオ構築・ベットサイジング・構造変化検知にある。

## 9. 実装必須チェックリスト（本番投入前ゲート）

- [ ] **分数階差分**：d=1のリターン化を排除し、定常性とメモリを両立したか
- [ ] **情報バー**：時間バーを排しドルバー（/インバランスバー）を採用したか
- [ ] **トリプルバリア**：ストップアウトを含むパス依存ラベリングか
- [ ] **メタラベリング**：一次予測にPrecision向上層を重ねたか
- [ ] **サンプル独自性**：非IID重み付けを適用したか
- [ ] **パージング/エンバーゴ**：リークを遮断したか
- [ ] **CPCV**：単一パスでなく分布で堅牢性を評価したか
- [ ] **DSR**：全試行を自動記録し、試行数で過学習判定したか
- [ ] **DAG（因果用途時）**：交絡/合流点を峻別し開示したか
- [ ] **アンサンブル**：単一故障点を回避したか

### クオンツへの3つの問い（デューデリジェンス）
1. バックテスト過学習をどう制御？（多重比較調整・感度分析・エンバーゴ）
2. このアノマリーの因果メカニズムは？（反事実的根拠）
3. アンサンブルを採用しているか？（単一故障点回避）

### 投資家が運用者に問うべき4点
1. 結果に至るまで合計何回試行したか
2. 試行間の相関は
3. 提示値は全試行の平均かベストか
4. 研究ログは存在するか
