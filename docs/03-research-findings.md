## 0. エグゼクティブ・サマリー

- **目的の転換（合意済み）**: 「いますぐ効く投資戦略を見つける」ことではなく、
  **「任意の戦略アイデアを、蓄積データと検証メソッドで効率的・確実に検証するシステム
  （＝検証ファクトリ）」を構築する**こと。**platform 自体が成果物**。
- **研究の結論**: crypto・日本株・ファンダ・信用空売り・指数・フロー・決算イベントまで
  広範に検証した結果、**`value`（割安, book/market）が唯一の“耐久性のある”候補**。
  他（size, short_to_long, gap, calendar, pairs, run-up, short-interest, flow→TOPIX）は
  **生存者バイアスのアーティファクト、もしくはエッジ無し**と判明。
  **フレームワークは偽アルファを一切作らなかった**（López de Prado の規律の実体化）。
- **最良の結果**: `value+PEAD` 合成が **DSR 0.87（プログラム最良）**。ただし認定基準
  0.95 には未達で、保留OOSでは PEAD が崩れ value 依存。
- **次段**: `value` を Phase 2（実運用）の“種”として小口・監視付きで展開。

## 1. 検証ファクトリ（platform）— 4要件の充足

| 要件 | 実体（`invest_system/research/` ほか） |
|---|---|
| ①戦略非依存の判定器 | `judge.py`：全試行を**永続グローバル登録**＋**デフレートDSR**＋PSR/minTRL/サブ期間/容量/最大DD→判定レポート(md/HTML) |
| ②戦略記述（IF＋糖衣） | `strategy.py`：`Strategy`契約＋7アーキタイプ（後述）。PIT `AsOfView` でリーク遮断 |
| ③コアデータ網羅 | 価格・財務・信用空売り・指数・投資部門別フロー |
| ④差分更新 | `data/updater.py`：`update`一発で新規のみ取得・冪等・再開可 |

### 偽陽性を一貫排除する設計（“判定器をp-hackできない”）
1. **先読み排除（構造）**: `AsOfView.asof(t)` は「t以前」しか返さない。戦略作者の善意に
   依存しない。生存者バイアスは**ポイントインタイム・ユニバース**で除去。
2. **選択バイアス（グローバル試行管理）**: 全試行を `TrialRegistry`（SQLite, 永続）に
   scope 単位で記録。**DSRは scope の累計試行 K でデフレート**。パラメータ格子の各点も
   独立試行。**再実行は指紋UPSERTで冪等**（Kを水増しできない）、新変種はKを増やし全戦略の
   基準を上げる＝**「通るまで回す」が効かない**。
3. **現実性**: 取引コスト・**執行ラグ（同足クローズの先読み排除）**・**容量（ADV制約のAUM
   上限）**を一級市民として評価。
4. **保留OOS**: 直近期間を分けて持続性を確認。

テスト 180 件（純関数中心、先読み排除・冪等性・デフレートを固定）。

## 3. 研究知見 — 全試行の判定（永続レジストリ 12 scope / 52 試行）

| 戦略ファミリー | アーキタイプ | 最良DSR | 結論 |
|---|---|--:|---|
| **value（割安）** | クロスセクション | **0.79–0.84** | ⭐**唯一の耐久候補**。PSR0.87・OOS持続 |
| **value+PEAD 合成** | クロスセクション | **0.87** | ⭐最良だが未達・OOSはvalue依存 |
| PEAD（予想改訂） | クロスセクション | 0.65–0.72 | IS有望・**OOS脆弱**（補助） |
| short_interest | クロスセクション | 0.09 | 空売り残高多＝ショート勢が正・逆張りは弱 |
| gap reversal | イベント駆動 | 0.21–0.25 | 偽陽性デフレートの教科書例 |
| flow→TOPIX | 時系列タイミング | 0.28 | 同時相関は強いが予測力弱 |
| calendar（月末効果） | 時系列タイミング | 0.38(SR負) | 本期間で効果消失 |
| pairs（トヨタ/ホンダ） | ペア/相対価値 | 0.02 | スプレッドはトレンドし回帰せず |
| earnings run-up（日次） | 日次イベント窓 | 0.21 | 正しく実装してもエッジ無し |
| size（PITユニバース） | クロスセクション | 0.35 | **生存者バイアスで偽DSR0.82→0.35に崩壊** |
| short_to_long（信用） | クロスセクション | 0.24 | 同上（偽0.68→0.24） |

### 最重要の発見：生存者バイアスが“偽の本命”を2つ生んでいた
固定ユニバース（全期間流動性で選抜）では **size(DSR0.82)・short_to_long(0.68)** が最有力に
見えたが、**ポイントインタイム・ユニバース**（各時点の流動性で時変選抜）に変えると両者は
**崩壊（0.35 / 0.24）**。＝厳密な方法論が、存在しない2つのエッジへの賭けを未然に防いだ。

## 4. 耐久性のあるコア：`value`

- **survivorship 補正後も生存**し、**保留OOS(2024-)でも持続**（OOS SR +0.49 ≥ IS +0.32）、
  **PSR(真SR>0)=0.87**。＝全プログラムで最も信頼できる信号。
- だが **年率Sharpe ~0.36 は低く、minTRL ≈ 241か月（~20年）**＝**単独では認定（DSR≥0.95）
  に必要な履歴が不足**。10年では「本物らしいが未証明」。
- **value+PEAD 合成**は相関 −0.08（ほぼ独立）で分散が効き、**DSR0.87・最大DD −27.4%（value
  単独 −43.6%より浅い）**＝最良。ただし **PEADがOOSで崩れ**、合成のOOSは +0.06 に失速。
  **OOS耐久性では value 単独が勝つ。**

**＝実運用の中核は `value`。PEAD は「効く局面だけ使う監視付き補助」。**

## 5. `value` を Phase 2（実運用）の種に

### 推奨する実運用形（“likely-real-but-unproven”として小口で）
- **戦略**: value（book/market）クロスセクション、**ポイントインタイム・ユニバース上位300**、
  **セクター中立**、月次リバランス、ロングショート（または long-tilt＋指数ヘッジ）。
- **容量**: 約 ¥6,500万（最も薄い建玉が10%ADVに達するAUM上限）→ 小口運用には十分。
- **ロールアウト**: 設計 D2 の通り **小口（〜¥50万）＋ライブ監視**で track record を積み増し、
  DSR が基準に近づくかを実データで確認（事前登録＋月次再校正＝設計 D3）。

### 配管（Phase 2）の留意点 — 設計の更新
- 研究は **暗号資産→日本株(value)** に着地。よって**執行は日本株ブローカー**が必要
  （元設計 D1 の bitbank＝暗号・公開データ専用とは別系統）。**未決の Phase3 ブローカー
  （IBKR か 国内ネット証券）をここで決定**する。
- 必要モジュール: 発注/OMS・ポジション/リスク管理・約定/コスト記録・監視。検証ファクトリの
  `Strategy`／エンジン／レジストリをそのまま“紙トレ→小口実弾”の判定ゲートに使える。

## 6. 正直な限界と今後

- **単一の 0.36-Sharpe 因子は、現有データ（10年）では認定不能。** これは日本株ファクター
  投資の現実であり、フレームワークの欠陥ではない。実運用しながら履歴を積むのが筋。
- **拡張余地**: Light/Standard→より長い履歴、Premium（売買内訳=日次需給・先物=ヘッジ/レジーム・
  /fins/details=フォレンジック会計）、外部ニュース/EDINETテキスト（NLP）。
- **研究の継続**は p-hacking 化リスクと隣り合わせ。**判定器の永続デフレートDSR**が、新たな
  試行のたびに基準を上げて自制を強制する。
