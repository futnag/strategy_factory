## 1. エグゼクティブ・サマリー

- **本プロジェクトの成果物は「検証ファクトリ」**である。すなわち、**任意の投資戦略アイデアを、
  蓄積データと統計的検証メソッドで“偽陽性を排除しながら”厳格に判定するシステム**。
  「いますぐ儲かる戦略を見つける」ことではなく、「アイデアを確実・効率的に検証する装置」を
  作ることが、合意した目的である。
- **広範な探索（暗号資産・日本株・財務・信用空売り・指数・投資部門別フロー・決算イベント）の末、
  唯一の耐久的な候補は `value`（割安, book-to-market）**だった。他の有望に見えた信号
  （size, short_to_long など）は**生存者バイアスのアーティファクト**、もしくはエッジ無しと判明。
  **フレームワークは偽アルファを一度も生成しなかった。**
- **最終的な最有力候補**は、PEAD（決算予想改訂）の OOS 失敗を**診断して経済的に修正**した
  **`value ＋ ロングティルト PEAD` の合成**で、**DSR 0.91（プログラム最良）・OOS Sharpe +0.62・
  最大ドローダウン −19.2%・容量約¥6億**。ただし認定基準 DSR≥0.95 には未達で、修正が
  OOS情報を用いた点（後述の留保）がある。
- **次段は Phase 2（実運用配管）**。研究はこれ以上の在サンプル精緻化が過学習と隣接する地点に
  到達しており、**真の検証は“フォワード（小口の実運用＋監視）”でしかできない**。

## 2. 経緯 — 目的の変遷と研究の流れ

### 2.1 出発点：López de Prado の金融機械学習
添付された10文書（学術論文＋日本語ブリーフィング）を統合し（`docs/01`）、その要点を備えた
投資システムを構築するのが当初の依頼だった。中核思想は **「過学習・自己 p-hacking こそ
最大の敵」「方向予測より、ボラ・サイジング・相対価値に ML を使う」「因果的に妥当な特徴量」
「全試行を記録し DSR で誠実に評価する」**。まず検証ハーネス（DSR/CPCV/パージング/試行
レジストリ）とデータ層（情報ドリブン・バー、分数階差分）を実装・テストした。

### 2.2 暗号資産での探索と帰結（bitbank BTC/JPY）
- **方向予測にエッジ無し**：4時間足・素朴特徴／真のドルバー／マイクロ構造特徴のいずれでも
  **DSR ≈ 0.2–0.4**（有意でない）。「OHLCV 特徴を足しても本物のα探索には不十分」と確認。
- **ボラは予測可能**：purged-CV の OOS で **将来ボラ R²=0.256**（将来リターン方向は R²=−0.02）。
  López de Prado の主張をデータで再現。ただし単純な vol-targeting では Sharpe 改善に繋がらず
  （実現ボラは 22%→6% に安定化したが、Moreira-Muir のα非有意）。
- **相対価値（クロスセクション）が最も有望だが非有意**：15 JPYペアの long-short ファクターで
  生 Sharpe 0.57 だが **DSR=0.60**（試行数 K=6 補正後）。多因子合成も単独最良を超えず。
- 結論：暗号資産では簡単なエッジ無し（収穫逓減）。**柱C（因果ファクター投資）の本来の土俵
  ＝日本株＋ファンダ**へ転換。

### 2.3 日本株・ファンダへの転換（J-Quants）
- データ源を **J-Quants V2 API**（APIキー方式）に。当初無料枠で検証し、**`/fins/summary`
  （決算サマリー）が無料枠で取得可能**（実績＋予想＋開示日 DiscDate＝バリュー/クオリティ/
  サイズの材料が揃う）と判明。詳細BS/PL/CF の `/fins/details` は Premium 限定。
- **レート制限**（無料5/分・大幅超過で5分遮断）を実装で自衛（ペーシング＋バックオフ）。
- ユーザーが **Standard プラン（120/分・10年履歴）契約**。10年・全銘柄の取得が現実的に。
- **固定ユニバース（全期間流動性で上位300選抜）での10年クロスセクション研究**：
  最良は size（小型株効果）SR 1.10 / **DSR 0.82**、value は弱く(+0.09)、quality はマイナス、
  素朴合成 COMPOSITE は −0.30。**サブ期間分析**で size は 2016-19 に+3.07→直近+0.05 と
  減衰が見えた。

### 2.4 「生存者バイアス」の発見 — 研究の転回点
ユーザーの「10年前のデータ（HFT前）に意味があるか？」という非定常性への問いを受け、
**時間減衰の重み付け**と**サブ期間安定性検定**を実装。さらに**ポイントインタイム・ユニバース**
（各時点の trailing 流動性で時変選抜＝先読み・生存者バイアスを排除）に切り替えたところ、
**結論が覆った**：

| factor | 固定300（バイアス有） | PITユニバース（補正後） |
|---|---|---|
| size | SR +1.10 / **DSR 0.82** | SR −0.33 / **DSR 0.35**（崩壊） |
| short_to_long（信用） | \|SR\| 1.14 / **DSR 0.68** | −0.22 / **0.24**（崩壊） |
| **value（book/market）** | +0.09 / 弱 | **+0.36（直近+0.87）/ DSR 0.38**（生存・強化） |

**＝最も有望に見えた2信号（size, short_to_long）は生存者バイアスの産物**で、厳密な方法論が
存在しないエッジへの賭けを未然に防いだ。**value だけが補正後も生き残った。**

### 2.5 目的の再定義：検証ファクトリ
ユーザーが方向性を明確化：「いますぐ効く戦略探しではなく、**思いついた戦略を蓄積データと
検証メソッドで効率的・確実に検証するシステムを作る**こと」。4要件（①戦略非依存の判定器
②戦略記述テンプレ ③コアデータ網羅 ④差分更新）を合意。私の批判的提案として
**「判定器の価値はグローバル試行管理＋事前登録＋PIT構造化が9割。DSLは罠（コードIF優先）。
リーク防止はフレームワークが所有。全DLは過剰（コア網羅＋オンデマンド）。現実性を一級市民に」**
を提示し、**MVP判定器を先に／コードIF＋ルール糖衣／コア網羅＋オンデマンド**で合意。

### 2.6 検証ファクトリの構築と戦略の検証
MVP（PITデータビュー→戦略IF→エンジン→判定器）を構築（§4）。続いて多様な戦略アーキタイプ
を判定器にかけた（§6.4）。さらに二大候補 `value` と `PEAD`（予想改訂）を深掘りし、
**PEAD の OOS 失敗を診断→ロングティルト化で修正**し、最有力候補（§7）に到達。研究フェーズを
本レポートで確定する。

## 3. 用語集 — 重要概念の解説

本プロジェクトの判断はすべて以下の概念に依拠する。

### 統計・検証
- **Sharpe比（SR）**: 平均超過リターン ÷ リターンの標準偏差。リスク1単位あたりの収益。
  年率Sharpe = per-period Sharpe × √(年間期間数)。月次なら ×√12。
- **PSR（Probabilistic Sharpe Ratio, 確率的シャープ比）**: 観測 Sharpe が与えられたとき、
  **真の Sharpe がベンチマーク（通常0）を超える確率**。標本長・歪度・尖度（非正規性）を補正。
  「この Sharpe は本当に正か？」を確率で答える。
- **E[max SR]（期待最大シャープ／False Strategy 定理）**: 真の Sharpe が全て0の N 個の独立な
  試行から得られる**最大 Sharpe の期待値**。試行を増やすほど「偶然の最良」が大きくなる。
  López de Prado & Lewis (2018)。**多重検定の罠の定量化**。
- **DSR（Deflated Sharpe Ratio, デフレート・シャープ比）**: PSR をベンチマーク=E[max SR] に
  して計算した値＝**「多重検定・非正規性・標本長を補正した“真の Sharpe>0”確率」**。
  本プロジェクトの**最終判定指標**。DSR≥0.95 を「多重検定後も有意（PASS）」とする。
  **試行（K）を増やすほど基準が上がる**ため、人が「通るまで回す」ことを封じる。
- **minTRL（minimum Track Record Length, 最小実績長）**: 観測 Sharpe を 95% 信頼で「正」と
  認定するのに必要な最小観測数。低 Sharpe ほど長い履歴が要る（例：年率0.36なら約241か月≈20年）。
- **IC（Information Coefficient, 情報係数）**: 各時点で「ファクター値 と 翌期リターン」の
  クロスセクション順位相関。**シグナルの予測力**を測る。正で安定なら有効。
- **IS / OOS（In-Sample / Out-Of-Sample）**: 設計・検証に使う期間 / 設計に使わず取っておく
  検証期間。OOSで持続するかが頑健性の証拠。
- **purging（パージング）/ embargo（エンバーゴ）**: 学習と検証の標本が時間的に重なって
  情報が漏れるのを防ぐ除外処理。**CPCV（Combinatorial Purged Cross-Validation）**は
  パージング付きの組合せ交差検証で、複数の train/test 分割（パス）で戦略を評価しレジーム
  依存を露出させる。

### データ・構築
- **ポイントインタイム（PIT）整合**: 各意思決定時点 t で「t 以前に実際に利用可能だった情報」
  のみを使うこと。**先読み（look-ahead）バイアス**＝未来情報の混入を断つ最重要規律。
- **生存者バイアス（survivorship bias）**: 「最後まで生き残った／流動的だった銘柄」だけで
  分析すると、敗者・退場銘柄が抜け落ち、成績が過大評価される歪み。本プロジェクトでは
  **固定ユニバース選抜**がこれを生み、**PITユニバース**（時変選抜）で除去した。
- **セクター中立化（sector-neutral）**: 各時点で同一業種内の平均を引き、業種という共通要因
  （交絡因子）を除いてファクター固有の効果を取り出す。
- **分数階差分（FFD）**: 系列を定常化しつつ「記憶（自己相関）」を最大限保つ前処理
  （López de Prado）。整数階差分（リターン化）は記憶を捨て過ぎる。
- **情報ドリブン・バー（ドルバー等）**: 一定時間でなく一定の売買代金ごとにバーを刻む。
  実データで超過尖度 5.27→0.97 等、**IID正規に近づく**（統計的性質が改善）。

### 戦略・因果
- **クロスセクション・ファクター**: ある時点で全銘柄を指標でランクし、上位ロング/下位ショート。
- **ロングショート / ダラーニュートラル**: 買いと売りを同額に組み、市場全体の動き（ベータ）を
  相殺して銘柄選択の超過収益（α）を取り出す。
- **value（バリュー）**: 割安株（高 book-to-market＝純資産/時価総額が大）が割高株を上回るという
  リスク/行動プレミアム。**PEAD（Post-Earnings-Announcement Drift）**: 決算・予想改訂後に
  株価が漸進的にドリフトする現象（本プロジェクトでは会社予想の改訂率を用いた）。
- **メタラベリング（corrective AI）**: 一次モデルの「方向」に対し、二次モデルが「いつ・
  どれだけ賭けるか（サイズ）」を学習する補正層（López de Prado ch.10）。
- **交絡因子（confounder）/ 合流点（collider）**: 因果ダイアグラム上、制御すべき変数
  （X←Z→Y）と、制御してはならない変数（X→Z←Y）。合流点を回帰に入れると偽相関が生じ
  推定符号が反転しうる。本プロジェクトの因果フィルタ（pairwise LiNGAM）はこれを判定する。
- **容量（capacity）**: 市場インパクトを許容範囲に保ったまま運用できる資金規模の上限。
  ここでは「最も流動性の薄い建玉が participation×ADV（平均売買代金）に達する AUM」と定義。

## 4. 検証ファクトリ（platform）— 技術的詳細

`invest_system/research/` が本体。設計思想は **「人が判定器自身を p-hack できないこと」**。

### 4.1 全体アーキテクチャ（層）
```
データ層(data/) ── 差分更新 ──▶ ローカルParquet
        │
        ▼  PITデータビュー(AsOfView) ＝ 戦略に「基準日以前」しか見せない
戦略(Strategy契約) ── target_weights(asof) ──▶ 目標ウェイト
        │
        ▼  エンジン(engine) ＝ コスト・執行ラグ・容量込みで損益系列化
判定器(judge) ── デフレートDSR + 永続レジストリ + PSR/minTRL/サブ期間/容量 ──▶ 判定レポート(md/HTML)
```

### 4.2 PITデータビュー（`data_view.py`）— リーク遮断の核
`AsOfView.asof(t)` は構築時に全パネルを「日付 ≤ t」へスライス済みの `AsOf` を返す。
戦略は `AsOf` からしかデータを読めず、**未来のバーを参照する術が構造的に存在しない**。
＝戦略作者の善意に依存せず、フレームワークが先読みを物理的に不可能にする。

### 4.3 戦略インターフェース（`strategy.py`）— DSL でなくコード契約
`Strategy.target_weights(asof) -> Series(銘柄→比率)` の一契約。実装済みアーキタイプ：
- `GapReversal`（イベント駆動：ギャップ後）、`CrossSectionalStrategy`（ランク→LS、`long_only`で
  市場ヘッジ・ロングティルト可）、`SignalTimingStrategy`（外部シグナル符号で単一銘柄）、
  `CalendarStrategy`（暦日条件）、`PairsStrategy`（2銘柄スプレッドz平均回帰）、
  `EarningsRunup`（日次イベント窓）、`CompositeStrategy`（複数戦略をウェイト合成）。
- 宣言的ルールは薄いクラスで“糖衣”化。複雑戦略（ニュース等）も同契約を実装すれば差せる。

### 4.4 バックテスト・エンジン（`engine.py`）— 現実性を一級市民に
- 各 t で `target_weights` を得て、t→t+1 の実現リターンと内積＝損益。空シグナル日は現金。
- **取引コスト**：回転率 sum|Δw| に片道bpsを乗じて控除。
- **執行ラグ（execution_lag）**：0=決定足の終値で執行、1=翌足で執行＝**観測した終値で建てない**
  （同足クローズの先読みを排除する現実寄り）。
- **容量（capacity_jpy）**：ADVパネルを与えると「最も薄い建玉が participation×ADV に達する
  AUM上限」を算出。**“紙上は効くが規模を持てない戦略”を露出**。
- 年率係数は日付間隔から自動推定（日次≈252, 月次≈12）。

### 4.5 判定器（`judge.py` ＋ `validation/`）
- `judge_grid(strategies, ...)`：戦略群（=1アイデアのパラメータ格子）をまとめて裁く。各点を
  `TrialRegistry` に**事前登録＋結果記録**（仮説と経済的合理性が必須＝a priori 理論の強制）。
- **DSR は scope の累計試行 K と Sharpe 分散でデフレート**。**パラメータ格子の各点も独立試行**
  ＝1つのアイデアでも閾値・保有・ユニバース等で数十試行に化けることを正しく計数。
- **永続グローバル・レジストリ**（SQLite, `data/research_trials.db`, gitignore）：試行は
  セッションを跨いで累積。**指紋（scope＋戦略＋params）UPSERT で再実行は冪等**（Kを水増し
  できない）、新変種は K を増やし**全戦略の基準を上げる**＝「通るまで回す／黙って試行を増やす」
  が効かない。
- 併せて **PSR・minTRL・サブ期間安定性・回転率・最大DD・容量** を報告し、**PASS/FAIL** を
  md / **HTML（インラインSVGのエクイティ曲線, 依存追加なし）** で出力。

### 4.6 偽陽性排除の二大柱
1. **先読み（構造）**：AsOfView ＋ PITユニバース。
2. **選択バイアス（永続デフレートDSR）**：グローバル試行管理。
加えて **現実性**（コスト・執行ラグ・容量）と **保留OOS** で多面的に弾く。テスト 208 件で
これらの性質（先読み排除・冪等・デフレート・容量算出）を固定。

size が最良に見えた（DSR0.82）が、後述の通り生存者バイアスの産物。

### 6.3 生存者バイアスの発見（最重要・§2.4 再掲）

#### gap 戦略に見るデフレートの威力（具体例）
12通りの格子で、生の最良は SR+0.78 / **PSR 0.98**（98%“正”に見える）だが、**12試行で
デフレートした DSR=0.25** → FAIL。素朴なバックテストなら勝ち戦略と誤認するところを、
判定器が best-of-12 のチェリーピックを却下した。

### 6.5 多ファクター breadth — 全ユニバース検証（実行待ち：全件ミラーDL後）
Grinold の能動運用の基本法則 **IR ≈ IC × √breadth** に基づく拡張。相関の低い経済的プレミアを
増やして「独立な賭けの数（breadth）」を稼ぐ方針＝**速度（ティック/HFT）でなく幅（日次ファクター）
で勝つ**。ソロ運用の比較優位は低頻度側にある。
- **シグナル**：value(B/M)・momentum(12-1)・quality(ROE＋**accruals＝利益の質**)・**low_volatility(低ボラ)**。
  low_volatility と accruals は今回追加（`equities/factors.py`）。配当利回りも `/fins/summary` 内包で利用可。
- **全ユニバース組立**：`equities/fundamentals.py` の `fundamentals_panel()` ＝ `load_fundamentals`
  （`fins_summary/` 全件 by-date ミラーを長形式化、旧 by-code も併合・重複除去）→ `point_in_time`
  （DiscDate≤t−lag）で先読みなし as-of。研究例は銘柄別ループから本経路へ移行済み（全11本）。
- **検証**：`examples/research_breadth_factors.py`（単一4ファクター＋合成を判定器で裁定、TOP_N=500）。
  **永続レジストリの試行数K増加で DSR 閾値も上がる**ため、合成がSR最良でも DSR は別要件＝幅を
  増やしても偽陽性防止は緩めない。**実測は全件ミラー（fins_summary / daily_quotes）DL完了後**。

### 6.6 時系列・統計的裁定（共和分ペア）— 実装・フル検証【柱D・Ernie Chan 補完】
平均回帰（スプレッドの定常性を収益化）を**第3の独立アルファ源（柱D）**として検証ファクトリに
実装した。予測（柱B）でも risk-premium 帰属（柱C）でもない別系統で、**市場中立・低相関**ゆえ
§6.5 の breadth（IR≈IC×√breadth）を補強する。原理は `docs/01`§11、設計は `docs/02`§5.2・DP12-14。
- **なぜ今か（既存の素朴 pairs の失敗を踏まえて）**：§6.4 の `pairs（トヨタ/ホンダ）DSR 0.02` は、
  **共和分を検定せず固定ヘッジ比で z 回帰した**素朴版の結果（「スプレッドがトレンドし回帰せず」）。
  これは Chan 手法の不在そのものが原因で、(a) 共和分ゲート未実施 (b) 固定ヘッジ比 (c) 半減期に基づく
  保有期間設計の欠如、を補えば**検証可能な仮説に昇格**する。
- **構築（実装済み）**：`timeseries/` に 半減期(OU)/Hurst/分散比/CADF・Johansen・Kalman。
  戦略は `research/strategies_meanrev.py` の `CointegratedPairs`（素朴 `PairsStrategy` の昇格＝CADF
  ゲート＋AsOf 動的ヘッジ）・`JohansenBasket`・`LinearMeanReversion`。日足 wide は `panel.load_daily_panel`。
- **検証規律（非交渉）**：① **ペア探索は全候補を試行計上**（N(N−1)/2 ＝巨大な多重検定面・DP13）＝
  「共和分ペアを見つけた」を在サンプルの偶然から守る。② ヘッジ比/共和分は **AsOf 推定**（先読み厳禁・
  DP12、Kalman が自然に解決）。③ 保有期間・再校正は**半減期から導出**（DP14）。
- **即実行可能**：取得済みの**日足全銘柄ミラー＋業種マスタ(S33)**で、業種内ペアを追加DLなしにスキャン
  できる。判定は `judge_grid(scope="meanrev_pairs")` でコスト・執行ラグ・容量込み。
- **スクリプト（実装済み）**：`examples/research_meanrev_pairs.py`（業種内ペアを CADF ゲート→全候補を
  registry 計上→DSR デフレート）。`judge_grid(extra_trials=)` でスキャン総数を K に算入（DP13）。
- **フル検証（既定パラメータ・全期間 2016–2026・2439営業日）**：業種内 326 候補（34業種×流動性上位5）を
  走査→44 が形成窓(504日)CADF を通過、**全 326 を K に計上**（建玉44＋スキャン282・V[SR]=0.0003）。最良
  ペアでも **SR(ann)+0.23・DSR0.03（K=326 でデフレート）＝FAIL**、minTRL≈12000か月（事実上認定不能）、
  サブ期間も符号反転（+0.16→−0.31→+0.50＝不安定）。共和分ゲート＋動的ヘッジへの昇格でも**偽アルファは
  生成されず**（§6.4 の素朴 pairs DSR0.02・スモークと整合）＝**厳密な多重検定後に耐久エッジ無し**。意義は
  「検証ファクトリが第3アルファ源(柱D)でも正しく機能した」こと。インフラ確立済み＝今後は変種（Kalman
  ヘッジ・entry/exit 格子・半減期連動保有・Johansen バスケット）を同 scope の追加試行として同一規律で裁ける。

## 7. 最有力候補 — value ＋ ロングティルト PEAD

### 7.1 value（割安, book-to-market）— 耐久的なコア
- **生存者バイアス補正後も生存**し、**保留OOS(2024-)でも持続**（OOS SR +0.49 ≥ IS +0.32）、
  **PSR(真SR>0)=0.87**。＝全プログラムで最も信頼できる単独信号。
- だが **年率 Sharpe ~0.36 は低く、minTRL≈241か月（≈20年）**＝**単独では認定（DSR≥0.95）に
  必要な履歴が不足**。10年では「本物らしいが未証明」。

### 7.2 PEAD（会社予想の改訂）と OOS 失敗の解明
- value と PEAD は**平均クロスセクション相関 −0.08（ほぼ独立）**。等加重合成（旧 LS合成）は
  全期間 DSR 0.87・最大DD−27.4% と単独を上回ったが、**保留OOSで +0.06 に失速**。
- **診断（`research_pead_oos_diagnose.py`）**：
  - ① IC が **IS +0.010 → OOS −0.035** と反転（予測力が失われた）。
  - ② **脚分解が決定的**：上方修正の**ロング脚はOOSも堅調（超過 +2.74%→+3.57%）**、
    下方修正の**ショート脚が崩壊（−2.50%→+3.00%＝もはや売れない）**。
  - ③ value との相関が **−0.06→−0.12** と対立を強めた。
- **機構**：下方修正銘柄＝割安・ディストレス＝**value がロングする側**。2024-26 のバリュー
  復権レジームでこれらが反発し、**PEAD のショート（下方修正売り）が value のロング（割安買い）
  と正面衝突して負けた**。

### 7.3 修正：ロングティルト化（診断駆動）
- PEAD のエッジは**ロング側（上方修正）にあり堅牢**、ショート側が脆く value と対立する、
  という診断に基づき、**PEAD をロングオンリー（上位ロング × ユニバース等加重ショート＝市場
  ヘッジ・下位を名指しで売らない）**に変更し、`CompositeStrategy` で value(LS) と 50/50 合成。

### 7.4 最終結果と留保
| | DSR | OOS SR | 最大DD | 容量 |
|---|--:|--:|--:|--:|
| **value + ロングティルトPEAD** | **0.91** | **+0.62** | **−19.2%** | ¥6億 |
| value 単独 | 0.82 | +0.49 | −43.6% | ¥6,574万 |
| 旧 value+PEAD(LS) | 0.87 | +0.06 | −27.4% | ¥7,033万 |

- **OOS が IS より強い（+0.43→+0.62）**＝旧合成の失速を完全に解消。診断どおり、value と
  衝突する「下方修正ショート」を外しただけ。**DSR 0.91（プログラム最良）・最大DD −19.2%
  （value の半分以下）・容量約¥6億**。

**正直な留保（重要）**：
1. **この修正は OOS を見て設計した（post-hoc）**。よって OOS +0.62 は厳密には“真のOOS”では
   ない。**ただし経済的合理性は健全**（割安・ディストレス銘柄をショートで叩くと value と構造的
   に対立する、という機構の修正＝カーブフィットではない）。
2. **DSR 0.91 はこの scope 局所（K=3）**。全プログラムの試行でグローバルにデフレートすれば
   下がる。**なお 0.95 未達。**
3. **真の検証は“新データ＝フォワード（実運用＋監視）”でしかできない。**

## 8. 方法論が示したこと（教訓）

- **偽陽性は2つの源から来る**：先読み（リーク）と選択バイアス（提出までの試行数）。本システム
  は前者を **PIT 構造＋生存者バイアス除去**で、後者を **永続デフレートDSR**で封じた。
- **“立派に見える偽物”は本当に出る**：size/short_to_long の DSR0.82/0.68、gap の PSR0.98。
  いずれも厳密な方法論が偽物と見抜いた。**素朴なバックテストは危険**。
- **合成は「逆位相だと相殺、低相関＋同符号だと分散効果」**：value×short_to_long は逆位相で
  失敗、value×PEAD(ロングティルト)は低相関で成功。**機械的な因子合成は失敗（通算複数回）**、
  効くのは**経済的に筋の通った合成**だけ。
- **失敗の“理由”まで見えると修正できる**：PEADのOOS失敗を脚分解で「ショート脚が value と
  衝突」と特定し、経済的に修正して最良候補に。判定器の診断（IC・脚分解・相関・容量・回転）が
  鍵だった。

## 9. 正直な限界

- **単一の ~0.36 Sharpe 因子は、現有10年では認定不能**（minTRL≈20年）。これは日本株ファクター
  投資の現実であり、フレームワークの欠陥ではない。
- **最有力候補の OOS は post-hoc**（§7.4）。真の確証はフォワードが必要。
- **データの制約**：信用残高は全銘柄“週次”止まり、決算発表予定は履歴なし、売買内訳・先物・
  詳細財務は Premium、ニュースは外部。より長い履歴（Premium 20年）や日次需給で再検証の余地。
- **研究の継続は p-hacking 化リスクと隣接**。永続デフレートDSRが新試行のたびに基準を上げて
  自制を強制するが、最終的には**フォワードでの確認に勝る検証は無い**。
